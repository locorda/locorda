## The Vision

**Cloud-synced apps shouldn't mean giving up data ownership.**

Most "cloud-synced" applications today store your data on servers controlled by the developer, not by you. You gain convenience but lose ownership, privacy, and control.

**What if we could have both?**

Locorda explores a different approach, inspired by Tim Berners-Lee's **SOLID project**:

<div class="vision-principles">

### üîê User-Owned Data
Your data lives in **your** pod, under **your** control. Not on vendor servers.

### üîó Linked & Interoperable
Applications can connect data semantically using RDF, while keeping it private when needed.

### üåê Standards-Based
Built on W3C standards (RDF, SOLID) for long-term interoperability and vendor independence.

### ‚ö° Developer-Friendly
Type-safe APIs, code generation, and excellent IDE support make semantic web development productive.

</div>

## Quick Start

```bash
# Core RDF library
dart pub add locorda_rdf_core

# Type-safe vocabularies
dart pub add locorda_rdf_terms_schema

# Object mapping
dart pub add locorda_rdf_mapper
```

<div class="code-comparison">

**Parse any RDF format:**
```dart
import 'package:locorda_rdf_core/core.dart';

final graph = turtle.decode('''
  @prefix schema: <http://schema.org/> .
  <#alice> a schema:Person ; schema:name "Alice" .
''');

print(jsonldGraph.encode(graph));  // Convert to JSON-LD
```

**Type-safe vocabulary access:**
```dart
import 'package:locorda_rdf_terms_schema/schema.dart';

// No more string typos!
final nameProperty = SchemaPerson.name;  // IDE autocomplete ‚ú®
final emailProperty = SchemaPerson.email;
```
